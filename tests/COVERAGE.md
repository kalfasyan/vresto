# Test Coverage

This file tracks unit-test coverage for the `vresto` package.

> **The snapshot below is regenerated by [`scripts/update_coverage_report.py`](../scripts/update_coverage_report.py).**
> Whenever you add or change tests, run `make coverage-report` and commit the
> updated section in the same PR so the numbers don't drift.

## Strategy

Coverage configuration lives in [`pyproject.toml`](../pyproject.toml) under
`[tool.coverage.run]`. We:

- track **branch coverage** (a missed `else` is a missed branch, not a free
  pass);
- source from `src/vresto`;
- **omit** the following packages because they need an end-to-end harness
  rather than unit tests, and including them would make the headline
  number meaningless:
  - `src/vresto/_version.py` — generated at build time;
  - `src/vresto/cli/*` — Typer command surface, exercised via CLI smoke tests;
  - `src/vresto/ui/*` — NiceGUI views, exercised via browser-driven tests.

Test modules sit under `tests/`, mirroring the source layout:
`tests/api/`, `tests/bands/`, `tests/products/`, `tests/test_services/`,
`tests/ui/`.

## Known low-coverage hotspots

These modules are I/O-heavy and need substantial fixtures (moto for S3,
fake GDAL VFS, async tile-server harness) to test meaningfully. Numbers
will lag the rest of the codebase until those fixtures land:

| Module | Why it's hard to unit-test |
|---|---|
| `services/worldcover.py` | S3 mosaic fetch + COG generation |
| `services/lcm.py` | LCM raster streaming over /vsis3/ |
| `services/sentinel_stream.py` | Sentinel-2 TCI streaming over /vsis3/ |
| `products/products_manager.py` | boto3-heavy product / metadata ops |
| `products/downloader.py` | Concurrent band download orchestration |

## How to regenerate

```bash
make coverage-report
# or, equivalently:
uv run python scripts/update_coverage_report.py
```

The script runs the full test suite with coverage and rewrites the block
between the `coverage-snapshot:start` / `:end` markers in this file. It
exits non-zero if any test fails, so the snapshot can't be regenerated
from a broken suite.

To browse coverage line-by-line in a browser, run `make coverage-html` and
open `htmlcov/index.html`.
